PROBLEME : Votre base de donnees de production est VIDE
ERREUR : "la relation events n'existe pas"

================================================================================
                           METHODE ULTRA-SIMPLE
================================================================================

ETAPE 1 : Lancer le script
---------------------------

  Windows:  Double-cliquez sur OUVRIR-FICHIERS.bat

  Cela va ouvrir automatiquement :
    - Le fichier SQL dans votre editeur
    - Supabase SQL Editor dans votre navigateur


ETAPE 2 : Copier le SQL
------------------------

  Dans votre editeur de texte :
    1. Appuyez sur Ctrl+A (tout selectionner)
    2. Appuyez sur Ctrl+C (copier)


ETAPE 3 : Coller et executer
-----------------------------

  Dans Supabase SQL Editor (navigateur) :
    1. Appuyez sur Ctrl+V (coller)
    2. Cliquez sur le bouton "Run"
    3. Attendez 1-2 minutes

  NOTE : Si vous voyez "already exists" --> C'EST NORMAL !


ETAPE 4 : Deployer
-------------------

  Une fois les migrations terminees :
    - Lancez MAJ_DU_SITE.bat


ETAPE 5 : Tester
----------------

  1. Ouvrez votre site
  2. Connectez-vous en admin
  3. Verifiez que le badge "ADMIN" est visible
  4. Testez la creation d'un evenement

                            EN CAS DE PROBLEME
================================================================================

  Erreur "Failed to fetch"
    --> Utilisez OUVRIR-FICHIERS.bat au lieu du HTML

  Erreur "relation events n'existe pas"
    --> Les migrations ne sont pas encore appliquees (faites l'etape 1-3)

  Erreur "Supabase is not defined"
    --> Deployez la nouvelle version (etape 4)

  Erreur "permission denied"
    --> Connectez-vous en admin sur Supabase Dashboard
